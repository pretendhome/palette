## What This Is

Palette Peers is a governed local message bus for multi-agent coordination. It lets Kiro, Claude Code, Codex, and Perplexity discover each other, exchange structured messages, and enforce Palette's decision governance (ONE-WAY DOOR / TWO-WAY DOOR) at the protocol level.

Design principle: **transport is shared, execution is isolated.**

It is NOT a clone of claude-peers-mcp. It is NOT free-form agent chat. It is a governed envelope system with identity, risk classification, human checkpoints, and audit trails — built on HTTP + SQLite.

## Build History

Kiro designed the full architecture: ADR, protocol specs (envelope, capability registry, risk gates), SQLite schema, golden test fixtures, broker, and CLI. Got registration and peer listing working. Stalled on integration tests due to cascading issues (stale DB with foreign keys, fake PIDs getting cleaned, sender existence checks blocking test flow, non-self-contained test runner).

Codex picked up the stalled work and:

1. Rewrote `db.mjs` — added legacy FK schema detection + migration + `schema_meta` table
2. Fixed `index.mjs` — allowed reserved identities (`human.operator`, `system.broker`), enabled store-and-forward for unregistered recipients, fixed approved state transition (`approved` not `pending`)
3. Tightened `validate.mjs` — added thread_id/in_reply_to UUID validation, requires_ack boolean check, created_at/ttl_seconds validation, one_way_door state requirement, ack payload validation, canonical HandoffPacket validation
4. Aligned `execution_request` with canonical Palette `HandoffPacket` schema (backward-compatible with legacy payloads)
5. Rewrote `run_tests.py` as fully self-contained (spins up its own broker on port 17899 with temp DB, tears down after)
6. Added 7th test case: store-and-forward for unregistered recipients
7. Created `sdk-alignment-v1.md` documenting the mapping between peers envelope and Palette wire contract

## Known Gaps (Not Blocking v1)

### ~~No Adapters Yet~~ → RESOLVED
Claude Code MCP adapter shipped: `adapters/claude-code/server.mjs`
- 8 tools: peers_send, peers_fetch, peers_list, peers_status, peers_checkpoints, peers_approve, peers_reject, peers_thread
- Auto-registers with broker on MCP initialization
- Heartbeat every 15s
- Unregisters on shutdown (SIGTERM/SIGINT/stdin close)
- Wired into `.mcp.json` for Claude Code

### No `GET /capabilities` Endpoint
Listed in `capability-registry-v1.md` as part of the adapter contract but not implemented on the broker. Low priority — capabilities are returned in `list-peers` already.

### ~~No Systemd Service~~ → RESOLVED
Service installed: `~/.config/systemd/user/palette-peers-broker.service`
- Enabled + running
- Auto-restarts on failure (RestartSec=3)
- Survives reboots (WantedBy=default.target)

### ~~No TTL Expiry Test~~ → RESOLVED
Test 8 covers TTL expiry with past-dated created_at.

### ~~No Ack Flow Test~~ → RESOLVED
Test 9 covers send → fetch → ack lifecycle.

### CLI `send` Is Human-Only
The CLI `send` command hardcodes `from_agent: human.operator`. Fine for now — agents use adapters, not the CLI.

### Not Committed to Git
The entire `palette/peers/` directory is untracked on the `feature/palette-peers` branch. Needs a commit.

### ~~No `node_modules` in .gitignore~~ → RESOLVED
Added `node_modules/` and `package-lock.json` to root `.gitignore`.

### Remaining Adapter Gaps
- Kiro adapter not yet built (Kiro uses MCP servers via stdio)
- Codex adapter not yet built (Codex uses tools + sandbox)
- Perplexity adapter not yet built (API-only, poll-based)

### No HandoffResult Support
Agents cannot return results through the bus yet. execution_request is one-way. HandoffResult would need a new message_type or a structured ack payload.

## Architecture Alignment with Palette Core

| Palette Concept | Peers Implementation | Status |
|---|---|---|
| 🚨 ONE-WAY DOOR | `message_type: one_way_door` + `risk_level: critical` → held for human | ✅ |
| 🔄 TWO-WAY DOOR | `message_type: execution_request` + `risk_level: low` → pass | ✅ |
| Human checkpoint | `waiting_human` state + approve/reject endpoints | ✅ |
| Decision persistence | `gate_log` table (append-only audit trail) | ✅ |
| Agent identity | Stable `<agent>.<role>` format with declared capabilities | ✅ |
| Trust tiers | UNVALIDATED / WORKING / PRODUCTION with gate enforcement | ✅ |
| HandoffPacket | Embedded in `execution_request` payload, validated against schema | ✅ |
| HandoffResult | Not yet implemented in peers (agents don't return results through the bus yet) | ⚠️ Gap |
| Glass-box | All gate evaluations logged, all messages persisted, all decisions auditable | ✅ |
| Convergence Brief | Can be sent as `message_type: proposal` with semantic blueprint in payload | ✅ Design |
| ASSUMPTION label | Can be sent as `message_type: advisory` + `risk_level: medium` | ✅ Design |
| ⚠️ KNOWLEDGE GAP | Can be sent as `message_type: human_checkpoint` with retrieval plan in payload | ✅ Design |

## Risk Assessment

| Risk | Severity | Mitigation |
|---|---|---|
| Broker is single point of failure | Medium | SQLite WAL mode + message persistence means restart recovers state. Systemd service would auto-restart. |
| No auth on broker HTTP endpoints | Low (localhost only) | Broker binds to 127.0.0.1 only. No remote access. |
| Stale peers with dead PIDs | Low | `cleanStalePeers()` runs every 30s, skips system PIDs |
| Schema drift between peers and SDK | Medium | `sdk-alignment-v1.md` documents the contract. Validator enforces HandoffPacket shape. |
| Legacy payload compatibility | Low | Transitional support documented. New adapters should use canonical format. |

## Broker Hardening (Claude Code pass)

Changes made by `claude.analysis` during ownership takeover:

1. **Body size limit**: Added 1 MiB `MAX_BODY_BYTES` guard on `parseBody` to prevent unbounded reads
2. **Stale peer grace period**: 60-second grace period after registration before PID cleanup (prevents PID reuse races)
3. **Graceful shutdown**: SIGTERM/SIGINT handlers close HTTP server and SQLite connection cleanly
4. **Test timestamp fix**: Store-and-forward test switched from hardcoded `created_at` to `new_iso()` (was silently expiring)
