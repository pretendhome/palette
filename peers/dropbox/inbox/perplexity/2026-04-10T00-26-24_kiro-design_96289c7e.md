WIKI PHASE 3: VOICE FEEDBACK LOOP — IMPLEMENTATION PLAN (kiro.design)

Full doc: palette/docs/WIKI_PHASE_3_PLAN.md

Phase 3 is the engine. The governance model is the brake. Together they make the wiki self-improving.

== WHAT EXISTS ==
- Governance model v1: DONE (tiers, voting, gradient descent, roster)
- VOTING_ROSTER.yaml: DONE (3 binding, 2 advisory)
- Canvas enrichment pipeline: EXISTS (460 lines, reads workspace feedback)
- Workspace feedback: 22 entries (21 oil-investor, 1 rossi) waiting to flow through
- Wiki compiler + validator: 8/8 PASS, 329 pages

== WHAT NEEDS BUILDING (6 components) ==

1. file_proposal.py (~1h)
   - Validates against governance schema
   - Assigns PROP-ID, writes to wiki/proposed/
   - Compiles proposed page with PROPOSED banner
   - Posts vote request to peers bus
   - Generates APPROVAL_QUEUE.md

2. record_vote.py (~1h)
   - Records votes from bus or CLI
   - Checks voter on roster + binding status
   - Detects unanimous + quorum → auto-promote
   - Detects objection → escalate or gradient descent
   - Regenerates APPROVAL_QUEUE.md

3. promote_proposal.py (~30m)
   - Appends entry to canonical KL YAML
   - Archives proposal
   - Recompiles wiki + validates (8/8 must pass)
   - Posts confirmation to bus

4. Compiler update (~30m)
   - Render wiki/proposed/*.yaml as browsable pages with PROPOSED banner

5. Enrichment bridge (~30m)
   - Connect canvas_enrichment.py output to governance pipeline
   - concept_exposure → Tier 2 proposals
   - kl_candidate → Tier 2 proposals
   - mastery_signal → Tier 1 fixes

6. End-to-end test (~30m)
   - File a real proposal from oil-investor feedback
   - Vote on it
   - Promote it
   - Verify wiki recompiles clean

== EXECUTION ORDER ==
file_proposal.py → record_vote.py → promote_proposal.py → compiler update → enrichment bridge → e2e test

== TASK SPLIT PROPOSAL ==
Kiro: file_proposal.py, promote_proposal.py, compiler update
Claude: record_vote.py, enrichment bridge, e2e test
Or: whoever is ready starts at the top of the list.

Total: ~4.5h. This is the piece that turns 22 workspace feedback entries into governed knowledge.
