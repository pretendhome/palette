PEERS BUS FIX — BROADCAST DELIVERY TRACKING (v1.1.0)

PROBLEM: Broadcast messages (to_agent=all) were never marked as delivered. Every fetch returned the entire history (390+ messages). New messages were buried. Agents saw the same broadcasts on every fetch.

FIX: Added per-agent broadcast delivery tracking.

Files changed:
- peers/migrations/002_broadcast_deliveries.sql — new table tracking which agent received which broadcast
- peers/broker/db.mjs — loads new migration
- peers/broker/index.mjs — /fetch and /peek now track broadcast deliveries per agent

Behavior:
- Direct messages (to specific agent): unchanged — marked delivered on fetch
- Broadcast messages (to_agent=all): now tracked per-agent. Each agent sees a broadcast exactly once. First fetch after the fix clears the backlog.
- /peek respects the same tracking (shows only unseen broadcasts)

Tested: 6/6 pass
1. Agent fetch 1 sees broadcast: PASS
2. Agent fetch 2 does NOT see same broadcast: PASS
3. Different agent sees same broadcast independently: PASS
4. Different agent fetch 2 does NOT see it again: PASS
5. New broadcast after backlog clear appears exactly once: PASS
6. Backlog clears after first post-fix fetch: PASS

Broker restarted. All 4 peers preserved. No data loss.

This is a TWO-WAY DOOR change — fully reversible by removing the broadcast_deliveries table and reverting the fetch logic.
