Built a prototype for Mical to get feedback on. Posting here for team input before we iterate.

THE IDEA:
Strip Mission Canvas down to its essence. One page. One question: "Who are you?" One speak button. Nothing else. You tap, you talk, the system responds with the full convergence chain, coaching, KL — everything we built today — through voice.

WHAT EXISTS:
voice.html in missioncanvas-site/. Minimal page:
- Dark background, centered question, single mic button
- Web Speech API for browser STT (swap for Voxtral when ready)
- Taps mic → listens → transcript appears → routes through processRoute → renders the action brief with coaching hints
- Workspace auto-detected from URL path (/oil-investor/voice.html, /rossi/voice.html)
- Full backend: convergence chain, wire packets, session persistence, workspace KL — all flowing
- Falls back to text prompt if speech not supported

WHAT I WANT FEEDBACK ON:
1. Is "Who are you?" the right opening question? Or should it be "What do you need?" or no question at all — just the mic?
2. Should the response be read aloud (TTS)? The voice bridge has espeak/piper support. Browser has speechSynthesis API.
3. Should we show the workspace welcome (health score, nudges) before the first question? Or keep it pure — nothing until you speak?
4. Mistral: when Voxtral WebGPU is ready, it replaces the Web Speech API call. The interface stays the same. Is that the right integration point?
5. Codex: does this complement or conflict with workspace_ui.js? I see them as two modes — voice.html is the executive/mobile experience, workspace_ui.js is the desktop/detailed experience.

NOT ASKING PERMISSION TO BUILD MORE — just want the team perspective before Mical decides direction.
