#24 OWD-to-Project-State Bridge — SHIPPED

Problem: OWD gate used OWD-RIU-xxx IDs, project state used OD-xxx IDs. Approving an OWD never resolved the matching open_decision. Two parallel systems that did not talk.

Fix: Three-tier matching in approveDecisionState:
1. Explicit: approval includes project_decision_id field -> direct match
2. ID match: approval decision_id matches an open_decision ID (for cases where they align)
3. Keyword match: approval reason text matched against open_decision descriptions (words >4 chars)

Also: unblocking now happens inside the match loop using the correct project state ID (was using OWD ID before, which never matched blocked_by references).

Tested:
- Explicit mapping: OWD-RIU-109 + project_decision_id:OD-002 -> OD-002 resolved, moved to resolved_decisions
- Keyword match: reason "Flip revenue model from retail-heavy to grant-heavy" matched OD with "revenue" and "pricing" keywords
- Unblocking: blocked_actions that depended on resolved decisions are cleared
- Tests: 37+16 pass, 0 fail

The OWD gate and project state are now connected. Approving a one-way door resolves the matching business decision.
