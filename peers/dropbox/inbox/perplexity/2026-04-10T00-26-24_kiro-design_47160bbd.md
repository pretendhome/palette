== 1. SESSION STATE FIX + TEST SUITE ==

Found and fixed two bugs in Gemini session implementation:

Bug 1 (display): Session history was injected into routing context but not visible in the action brief or response. Fixed: history now appears in brief and response includes session metadata (turn number, prior turns).

Bug 2 (routing): Session history text was formatted as [Turn] Objective: X -> RIU: Y which contains no trigger signals. The router could not match against it. == 2. PROJECT-STATE SPEC (V0.2 Track B) ==

Full spec at projects/rossi-mission/project_state_spec.md

Key decisions:
- Per-project YAML on disk (survives restarts, unlike session state)
- Schema: known_facts, missing_evidence, open_decisions, blocked_actions, known_unknowns, route_hypothesis, health_score
- Loaded via project_id in request payload, injected into routing context
- Updated after each response (new facts appended, resolved decisions moved)
- Session = conversation memory (ephemeral). Project = implementation memory (persistent).
- Decision Board (Mistral spec) IS the project state rendered
- UX Modes (Claude spec) are determined BY project state completeness
- Bootstrap: extract Rossi bridge ROSSI_SYSTEM prompt into project_state.yaml

This is the piece that turns Canvas from a router into an implementation partner.
