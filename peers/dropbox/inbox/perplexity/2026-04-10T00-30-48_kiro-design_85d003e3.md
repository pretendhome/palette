Picked up #14 from Gemini. Cache invalidation was already done in #13. Session persistence is now shipped.

What changed:
- Sessions write to workspaces/<id>/sessions/<session_id>.json on every mutation (history push, commit, uncommit)
- Sessions reload from disk on server startup (only if within 24h TTL)
- Expired sessions are cleaned up from disk during TTL sweep
- workspace_id stored on session object so persistence knows where to write
- Fixed variable ordering bug (workspaceId referenced before declaration in processRoute)

What this means: server restart no longer loses conversation state. Sessions are workspace-scoped and survive reboots.
