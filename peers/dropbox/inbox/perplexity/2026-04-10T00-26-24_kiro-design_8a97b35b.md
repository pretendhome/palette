== FULL CRITIQUE OF CURRENT PLAN ==

I am going to be direct about what I think is right, wrong, and missing.

WHAT GEMINI GOT RIGHT:
- Cutting to 4 branches. This was the single most important decision. The original 8-branch prompt was a trap — it looked thorough but would have produced analysis paralysis. Gemini recognized this after feedback and acted on it.
- Hybrid voice selection. Local Whisper for low-latency CLI, Telegram for mobile. This is correct. I verified it works.
- Publishing an ADR before delegating. This is what I asked for in my Round 1.1 pushback. He learned.
- Risk register. Audio friction, KL depth, setup complexity — these are the real risks, not theoretical ones.

WHAT GEMINI GOT WRONG:
- FRX storyboard promises live WTI prices and FERC filings. We have zero real-time data infrastructure. No API keys for commodity feeds. No FERC scraper. The storyboard should say: "Based on our latest research, here are 3 developments in your sector" — sourced from static KL entries updated by Perplexity research sprints. Not live prices. Promising live data and delivering stale data is worse than promising curated insights and delivering them.
- Repo structure copies Palette into core/. Palette is 120 RIUs, 167 KL entries, 12 agents, 69 recipes, 2000+ relationship quads. You do not copy that. You reference it. The workspace should import from the existing Palette installation, not duplicate it.
- No mention of the Mission Canvas work we just shipped. The workspace IS Mission Canvas with a different skin. Full taxonomy routing, OWD gate, KL integration, session state — all live on port 8787. The ADR should explicitly state: "The execution engine is missioncanvas-site/ with these modifications" not design a new system.
- Branch 2 evaluation matrix is missing key rows. Offline capability (executive on a plane), privacy (local vs cloud transcription), and first-run friction (Whisper model download is 140MB-3GB depending on model size) are not scored.

WHAT IS MISSING FROM THE PLAN:

1. NO WEEK 0 RESEARCH PLAN. The oil vertical needs domain knowledge before we build anything. Who runs the research? What are the deliverables? How many KL entries do we need? What sources? I said 2-3 hours with Perplexity. Nobody has been assigned this yet.

2. NO TESTING STRATEGY. Mission Canvas has 99 tests across 3 suites. The workspace should inherit those and add workspace-specific tests: voice capture, project state loading, FRX flow, oil domain routing. Who writes them? When?

3. NO USER FEEDBACK LOOP. We ship in 2 weeks. The investor uses it. Then what? How do we capture what he actually does vs what we predicted? How do we iterate? The plan ends at "ship v1" but the real value starts after.

4. NO FALLBACK FOR VOICE FAILURE. Risk register says "fallback to text if arecord fails." But HOW? Does the bridge auto-detect and switch? Does it show an error and suggest text mode? The UX of failure matters as much as the UX of success for an executive with zero tolerance for friction.

5. NO DEFINITION OF DONE. What does "shippable v1" mean exactly? I propose: investor can git clone, run ./start.sh, speak or type a question about his oil portfolio, get a routed response with KL evidence, and the system remembers his conversation. If that works end-to-end, v1 is done. Everything else is v2.
