# Artifacts By User Segment

**Author**: codex.implementation
**Date**: 2026-03-29
**Status**: Proposal

## Purpose

If the product is artifact-first, then different users should not receive the same default artifact set.

The engine can stay shared.
The default artifact mix should change by workspace.

## Segment 1: Small Business / Founder / Operator

Primary need:
- clarity
- prioritization
- funding readiness
- operational next steps

Best default artifacts:

1. **Decision Board**
- current mission state
- blockers
- open decisions
- next actions

2. **Funding Readiness Brief**
- what is missing
- what weakens the case
- what improves fundability

3. **30-Day Action Plan**
- concrete time-bounded sequence
- owner and operator separation

4. **Evidence Checklist**
- documents needed
- proof gaps
- priority order

5. **Stakeholder Update Draft**
- investor / advisor / partner-facing summary

6. **Business Plan Delta**
- what changed from current plan to stronger version

7. **Grant / Funding Tracker**
- deadlines
- requirements
- status

8. **Risk Summary**
- short list of business-critical unresolved risks

9. **Meeting Prep Brief**
- what to ask
- what to decide
- what evidence to bring

10. **Weekly Progress Digest**
- what changed
- what is still blocked
- what matters next

## Segment 2: Executive / Investor / Principal

Primary need:
- compressed orientation
- scenario awareness
- decision support
- stakeholder communication

Best default artifacts:

1. **Daily Market Brief**
- what moved
- why it matters
- implications for the portfolio or mission

2. **Decision Board**
- portfolio or mission state
- open decisions
- blocked actions

3. **Scenario Memo**
- best case
- base case
- downside case
- what changes the recommendation

4. **Regulatory Alert Brief**
- what changed
- relevance
- action implications

5. **Board / LP Update Draft**
- concise narrative for external stakeholders

6. **Position Tracker**
- current exposure
- current concerns
- unresolved evidence gaps

7. **Meeting Brief**
- prep for call, board, regulator, or partner discussion

8. **Evidence Pack**
- source-backed bundle for a live decision

9. **What Changed Since Yesterday**
- re-entry artifact for busy users

10. **Recommendation Note**
- recommended action
- why now
- what assumptions it depends on

## Segment 3: Analyst / Research / Strategy User

Primary need:
- synthesis
- comparison
- source handling
- hypothesis testing

Best default artifacts:

1. **Research Brief**
2. **Evidence Pack**
3. **Source Set**
4. **Comparison Table**
5. **Hypothesis Tracker**
6. **Known Unknowns List**
7. **Watchlist**

## Segment 4: Advisor / Consultant / Fractional Operator

Primary need:
- quick orientation into a client mission
- explainable recommendations
- reusable deliverables

Best default artifacts:

1. **Client Decision Board**
2. **Current State Summary**
3. **Gap Analysis**
4. **Recommendation Brief**
5. **Plan Draft**
6. **Evidence Request List**
7. **Client Update Draft**
8. **Meeting Follow-Up Draft**
9. **Decision Log Snapshot**
10. **Status Digest**

## Cross-Segment Rules

Some artifacts should exist in every workspace:
- Decision Board
- Project State
- Action Brief
- Evidence Pack
- Session Snapshot

Everything else can vary by workspace profile.

## Product Recommendation

Each workspace config should define:
- default artifact set
- startup artifact
- high-frequency artifacts
- role-specific artifacts

That lets the system feel tailored without splitting the engine.
