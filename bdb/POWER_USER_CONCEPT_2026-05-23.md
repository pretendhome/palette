# Power User Concept
## For Palette / Mission Canvas / BDB
**Date**: 2026-05-23
**Purpose**: define the `power user` as a product concept before narrowing to Claudia Canu as the first concrete example

---

## Why This Matters

Palette is not a mass-market consumer app.
It is a governed intelligence system that changes how a person thinks, decides, remembers, and teaches others.

That means adoption will not begin with everyone equally.
It will begin with a small number of people who:

- see the system's value earlier than others
- tolerate the initial complexity
- shape the workflows through real usage
- translate the system into the language of their organization
- become the proof that the product works

That person is the `power user`.

In enterprise software, Glean would often call this person a `champion`.
In Palette, the concept is deeper than internal advocacy.
The power user is not just the first enthusiastic adopter.
They are the first human whose judgment actually compounds inside the system.

---

## Definition

A `power user` in Palette is:

> a domain expert or high-agency operator who uses the system repeatedly enough, deeply enough, and reflectively enough that Palette becomes part of how they think, decide, and teach others.

This person is not defined by technical skill alone.
They are defined by behavioral depth.

They do five things:

1. Ask real high-value questions, not toy prompts.
2. Reuse prior decisions and improve them over time.
3. Notice the difference between generic AI output and governed judgment support.
4. Adapt the system to real work.
5. Become a bridge between the product and the next wave of users.

---

## Core Characteristics

A true Palette power user usually has most of these traits:

### 1. Domain confidence

They already know a lot.
They are not using Palette because they have no judgment.
They are using it because their judgment is valuable and they want it to scale, persist, and compound.

### 2. Repeated workflow ownership

They perform recurring work where better memory and better structure would clearly matter:

- curriculum design
- legal research
- compliance review
- strategic planning
- capability building

### 3. Pattern sensitivity

They naturally notice:

- repeated question shapes
- recurring failure modes
- reusable templates
- which decisions deserve to be remembered

### 4. Tolerance for early friction

They are willing to use a rough system before the product is polished because they can already see the underlying leverage.

### 5. Social credibility

They are trusted by peers.
When they say “this changed how I work,” other people listen.

### 6. Reflective practice

They do not just use outputs.
They review, refine, compare, and improve processes.
This is essential because Palette is a compounding system, not only a response system.

---

## What Makes A Power User Different From A Normal User

| Normal User | Power User |
|---|---|
| wants quick answer | wants reusable judgment |
| evaluates one interaction | evaluates how the system improves over time |
| may use the tool episodically | builds the tool into repeated workflows |
| sees convenience | sees structure |
| may adopt features | shapes product direction |
| can benefit | can compound |

The key distinction is:

> a normal user consumes the system; a power user co-evolves with it.

---

## What The Power User Does For Palette

The power user is not a side persona.
They are a core product mechanism.

They create value in four ways:

### 1. They prove the thesis

Palette’s thesis is not just that it answers well.
It is that it makes human capability compound.

The power user is the first visible proof of that claim.

### 2. They generate real memory

A power user creates the highest-quality memory objects:

- better decisions
- stronger reusable patterns
- cleaner workflows
- better feedback on what should be stored vs ignored

### 3. They expose product gaps

Because they push harder, they reveal:

- where the ontology is too hidden
- where the memory is too weak
- where the product surface is too rough
- where the domain pack is too thin

### 4. They become the internal teacher

They are the first person who can show someone else:

- what this is
- why it matters
- how to use it well
- what kind of behavior makes it powerful

This is why `power user` and `champion` overlap, but are not identical.
The champion advocates.
The power user also generates the best system behavior.

---

## Product Requirements For A Power User

If Palette wants power users, it must be designed for them deliberately.

A power user needs:

### 1. Traceability

They want to see:

- why the system classified something a certain way
- where an answer came from
- what prior decisions were connected
- what stayed local vs what went external

### 2. Memory continuity

They need to feel:

- that the system remembers their prior work
- that repeated usage makes future sessions better
- that the memory belongs to them, not the vendor

### 3. Configurable depth

They want more than a beginner UX.
They need:

- provenance
- confidence
- alternatives
- comparison
- history

This is already visible in the Claudia lens from the repo: expert users want confidence scores and provenance, not just simplified answers.

### 4. Reusable artifacts

They need the system to generate things that can be reused:

- briefs
- workflows
- playbooks
- decision logs
- curricula
- research structures

### 5. Teaching leverage

A power user wants to bring others in.
So the system must help them teach:

- what they did
- why it worked
- how someone else can repeat it

---

## What The Power User Needs From The UX

The UX for a Palette power user should feel like:

- a trusted workbench
- a memory-augmented reasoning environment
- a place where the system shows its work

It should not feel like:

- a toy prompt box
- a generic chat interface
- a black-box assistant

Power-user UX likely needs:

1. visible prior decisions
2. confidence + provenance views
3. connected-work indicator
4. artifact generation and reuse
5. domain-specific memory packs
6. controlled external research with explicit boundary

## The Power User In The BDB Submission

For the submission, the power user concept is strategically useful because it makes the product more believable.

Instead of pretending the product is already for everyone, we can say:

- Palette starts with power users in regulated domains
- the first buyer is not “everyone in a law firm”
- the first buyer is the trusted expert who figures it out first
- that user becomes the internal champion and workflow designer

This is a much stronger story than broad undifferentiated adoption.

It also fits the product honestly:

- the system is deep
- the ontology is meaningful
- the memory value shows up most clearly for repeated expert use

So the power-user wedge is not a weakness.
It is the correct entry path.

## The Product Principle

The power user should be able to say:

> This system does not just help me answer questions. It helps me build a better way of thinking, remember what mattered, and teach others how to do the same.

If Palette can produce that sentence honestly, the power-user concept is real.
