Why This Is My Angle

The core pattern in my work is structural. I look for what is mis-shaped underneath the learning problem, fix that environment, and then make the learner’s next move obvious enough that improvement becomes natural.

“I love creating environments where people succeed without realizing how they’re learning.”
From my Mnookin two-pager. That is the through-line from Paris teaching to Ask Pathfinder to Palette.
  1. Find the structural mistake. Do not start by adding more content.
  2. Design the right cue. Put the learning moment where the behavior actually happens.
  3. Ask for a real artifact. If nothing real is produced, the system taught too little.
  4. Adapt without becoming opaque. Coaching changes with the learner, but remains glass-box.
  5. Measure changed behavior. Not completion, not attendance, not satisfaction theater.

Honest boundary: this is a specimen, not a claim about Stripe’s current internal systems. The point is to show the architecture I would bring, using the same philosophy I have used in teaching, knowledge systems, and AI-native enablement.
