# Operator Briefing (Sales Lens, User-First)
Date: 2026-03-02
Audience: New operators joining Palette GTM + delivery motion

## 1) Deal Objective / Stage Context

Objective:
- Move from "AI curiosity" to a trusted production decision system that improves user outcomes.

Stage:
- Late validation / early scale.
- System is structurally healthy: integrity checks pass, regression stable, actionable traversal outputs are live.

## 2) Current Hypothesis (Why Deals Move or Stall)

Deals move when:
- Buyer sees a live, account-relevant workflow in minutes.
- User-level value is obvious ("this helps me do my job now").
- Risk/governance is explicit (Monitor decision contract, block routing, SLO checks).

Deals stall when:
- Conversation stays at architecture slides instead of user task flow.
- We pitch "AI platform" abstractly without showing one real decision packet.
- We fail to distinguish reversible experimentation from irreversible risk.

## 3) Qualification Gaps (Ordered)

1. User pain proof:
- Do we have one concrete recurring user workflow and current failure cost?
2. Decision owner:
- Who can approve operational adoption, not just technical trial?
3. Risk posture:
- Is this customer experimentation-forward or compliance-first?
4. Integration reality:
- Which data layers can be connected now vs phased?
5. Success metric clarity:
- What exact before/after metric will define trial success?

## 4) Stakeholder Map (Who Must Believe)

Primary:
- Daily user/operator (must feel immediate productivity gain).
- Technical owner (must trust reliability and observability).
- Economic buyer (must see credible ROI and reduced execution waste).

Common missing stakeholder:
- Governance/risk approver who needs explicit block paths and rollback logic.

## 5) Next-Step Plan (Owner + Date + Outcome)

1. Owner: Sales lead, Date: next call
- Outcome: Confirm one workflow + one user persona + one success metric.
2. Owner: Solutions/Engineer
- Outcome: Run live traversal for 3-5 RIUs matching customer workflow.
3. Owner: Product/Monitor facilitator
- Outcome: Show decision-state logic (`ship`, `ship_with_risks`, `ship_with_convergence`, `block`) with one example per state.
4. Owner: Customer champion
- Outcome: Name rollout cohort and adoption checkpoint date.

## 6) Objections / Risk and Response Strategy

Objection: "Why not just use generic LLM/copilot?"
- Response: Generic tools answer questions; Palette enforces cross-layer consistency, regression safety, and governed action routing.

Objection: "We can build this internally."
- Response: You can, but opportunity cost is high. Palette gives a production-ready trust stack now: integrity + audit + regression + drift + Monitor routing.

Objection: "AI changes too fast; this may be obsolete."
- Response: Palette is model/tool agnostic by design. Core value is not one model; it is decision trust over your enterprise context.

Objection: "This feels risky in production."
- Response: Reversible paths ship fast (`ship`, `ship_with_risks`). Irreversible/unsafe paths are blocked and routed to the correct agent (Debugger/Architect/Researcher).

## 7) Success Signals for the Next Milestone

- Customer can run one end-to-end workflow without assistance.
- At least one "before vs after" metric improves (time-to-decision, rework rate, escalation latency, or operator throughput).
- Stakeholders align on one governed rollout path (not just a demo wow moment).

## 8) Operating Principles (Lumen Perspective: Build for Users)

1. Demo reality, not promises.
- Show live user outcomes, not synthetic future-state decks.
2. Start from user task, not platform feature.
- If user value is unclear, deal quality is low.
3. Keep experimentation alive.
- Prefer reversible progress; do not over-block.
4. Separate evidence from opinion.
- Integrity/audit/regression/drift/Monitor outputs are the source of truth.
5. Sell what works now, with clear next gates.
- Confidence comes from proof + sequencing, not hype.

## One-Line Positioning

Palette is a user-first enterprise decision system: natural language in, governed action out, with structural proof at every step.
