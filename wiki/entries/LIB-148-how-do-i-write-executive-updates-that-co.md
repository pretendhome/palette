# How do I write executive updates that communicate AI project status without overwhelming leadership with technical detail?

RIU-105 structures executive updates in four sections, each designed for a specific executive question. Section 1 — Status (answers "Where are we?"): traffic light indicator (Green/Amber/Red) with a one-sentence summary. Example: "AMBER — Model accuracy meets target (92%) but data pipeline latency exceeds budget by 40%." Include progress against milestones as a simple percentage. Section 2 — Risks (answers "What could go wrong?"): top 3 risks, each with likelihood (High/Medium/Low), impact (High/Medium/Low), and mitigation status (Mitigated/In Progress/Unmitigated). Executives care about unmitigated high-impact risks — lead with those. Section 3 — Decisions Needed (answers "What do you need from me?"): list 1-3 decisions that require executive authority. For each: the decision, the options, the recommendation, and the deadline. If no decisions are needed, say so explicitly — executives appreciate not being asked to decide unnecessarily. Section 4 — Asks (answers "How can I help?"): specific resource, access, or escalation requests. Be precise: "Need VP Engineering to approve API access by March 15" not "Need more support." Format: the entire update should fit on one page or one Slack message. If it requires scrolling, it is too long. Frequency: weekly for active projects, biweekly for maintenance. Always send on the same day/time.
