# How can Generative AI assist in database migrations and schema refactoring?

**Proposal ID**: PROP-2026-04-04-009 | **Tier**: 2 | **Type**: new | **Evidence Tier**: 1

**Related RIUs**: RIU-500, RIU-200

Generative AI accelerates database migrations by automating schema mapping, SQL translation, and data validation. Key strategies include:

1. **Schema Mapping & Transformation**:
   - **Automated Entity-Relationship (ER) Extraction**: Use LLMs to analyze source schema DDL and generate target schema definitions, identifying equivalent data types and constraints.
   - **Semantic Column Mapping**: Identify semantically identical columns across disparate schemas (e.g., mapping 'cust_id' in legacy DB to 'account_uuid' in modern DB) using embeddings or LLM-based name synthesis.

2. **SQL Dialect Translation**:
   - Use fine-tuned models to translate legacy SQL (e.g., Oracle, PL/SQL) into modern dialects (e.g., PostgreSQL, BigQuery) while maintaining functional parity.
   - **Validation**: Automatically generate unit tests for translated queries to ensure identical results across source and target.

3. **Data Quality & Cleansing**:
   - **Synthetic Data Generation**: Generate high-fidelity synthetic test data that maintains the statistical properties of the source database for migration dry-runs.
   - **Automated Deduplication**: Use LLMs to identify and merge duplicate records that cannot be caught by simple primary key checks.

4. **Documentation & Knowledge Capture**:
   - Generate natural language documentation for legacy schemas based on table names, column comments, and stored procedures.

**Implementation Threshold**: Utilize AI-assisted migration tools when manual schema mapping or SQL translation exceeds 40 hours of engineering effort per microservice.
