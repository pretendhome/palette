# What is the difference between zero-shot and few-shot LLM performance, and when should I use each?

**Proposal ID**: PROP-2026-04-04-008 | **Tier**: 2 | **Type**: new | **Evidence Tier**: 1

**Related RIUs**: RIU-500, RIU-501

Zero-shot and few-shot learning describe how much context/example data is provided to an LLM during a prompt to enable a specific task.

1. **Zero-Shot Learning**:
   - **Definition**: The model is given a task and a prompt but zero examples of the desired output.
   - **When to Use**: For well-defined, general-knowledge tasks (e.g., 'Summarize this text') where the model's pre-training is sufficient.
   - **Benefits**: Minimum token usage, lowest latency, simplest prompt engineering.

2. **Few-Shot Learning**:
   - **Definition**: The model is provided with a small number (typically 1-5) of high-quality examples (input-output pairs) within the prompt.
   - **When to Use**: For complex, domain-specific, or highly structured tasks (e.g., 'Extract entities in this exact JSON schema') where the desired format or reasoning pattern is non-trivial.
   - **Benefits**: Significantly improved accuracy on specific tasks, better adherence to complex formatting rules, allows 'teaching' the model a specific tone or style.

**Comparison Table**:

| Metric | Zero-Shot | Few-Shot |
|---|---|---|
| **Accuracy** | High (General) | Very High (Specific) |
| **Token Cost** | Lowest | Higher (depends on example size) |
| **Latency** | Lowest | Slightly Higher |
| **Prompt Complexity** | Low | Medium |

**Implementation Threshold**: If zero-shot performance falls below your required accuracy threshold (e.g., <80%), implement 3-shot learning before moving to more complex techniques like fine-tuning or RAG.
